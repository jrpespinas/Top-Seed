# Screenshot Reference Notes

Store product and design inspiration screenshots here. These images are references only; they are not app assets.

Recommended folders:

- `queue-dashboard/`
- `check-in-flow/`
- `payments/`
- `player-status/`
- `leaderboard/`

For each group of screenshots, add a short notes file using this structure:

```markdown
# Reference Notes

## What Works

- The specific layout, interaction, or hierarchy worth learning from.

## What Does Not Fit

- Details that do not match badminton open-play sessions.

## Borrow

- The principle or behavior to adapt.

## Do Not Copy

- Visual details, branding, or layouts that should not be replicated directly.
```

Borrow principles, not pixels. Use references to sharpen hierarchy, interaction patterns, and component behavior before choosing colors or styling.
